Meta Conversions API (serverseitige Übermittlung)

Zusätzlich zum Meta-Pixel im Browser nutzen wir die Meta Conversions API. Dabei wird ein Konversionsereignis (z. B. eine erfolgreiche Terminbuchung) nicht nur vom Browser, sondern auch direkt von unserem Server an Meta übermittelt. Dies dient ausschließlich dazu, die Konversionsmessung verlässlicher zu machen, etwa wenn das Pixel im Browser durch einen Werbeblocker blockiert wird.

Browser-Pixel und Conversions API senden dasselbe Ereignis mit einer gemeinsamen Ereignis-ID; doppelte Übertragungen werden von Meta automatisch zusammengeführt (Deduplication).

An Meta übermittelt werden bei einer Conversions-API-Übertragung insbesondere:

  • Ereignisname und Ereignis-ID
  • Zeitstempel und URL der Seite, auf der das Ereignis ausgelöst wurde
  • IP-Adresse und User-Agent des Browsers
  • Sofern vorhanden: die Werte der Facebook-Cookies _fbp und _fbc

Die Conversions API wird ebenfalls ausschließlich nach Ihrer Einwilligung gemäß Art. 6 Abs. 1 lit. a DSGVO und § 25 Abs. 1 TDDDG aktiv. Sie ist an dieselbe Einwilligung gekoppelt wie das Meta-Pixel; ohne Einwilligung findet keine Übermittlung statt. Die Einwilligung können Sie jederzeit über den Link „Cookie-Einstellungen“ im Fußbereich widerrufen.
